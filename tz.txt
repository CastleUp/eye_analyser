1. Стек технологий
Язык: Python 3.10+

Детекция и Ландмарки: MediaPipe (модуль Face Mesh). Это быстрее и точнее для поиска глаз, чем каскады Хаара или YOLO.

Извлечение признаков (Embedding): InsightFace (модель Buffalo_L) или DeepFace (модель ArcFace). Эти модели отлично работают на малых VRAM.

Векторная база: ChromaDB (локальная, не требует отдельного сервера).

Вывод (UI): OpenCV для отрисовки кадра в реальном времени.

2. Архитектура решения
Этап 1: Захват и препроцессинг (Детекция)
Input: Стрим с веб-камеры через OpenCV.

Detection: MediaPipe находит 468 точек лица.

Cropping: По индексам точек извлекаем координаты левого и правого глаза с небольшим отступом (padding), чтобы захватить веки и форму разреза.

Normalization: Приводим кропы глаз к единому размеру (например, 112x112 или 224x224) и выравниваем по горизонтали.

Этап 2: Генерация эмбеддингов (Inference)
Model: Загружаем модель на GPU (Nvidia 6GB).

Embedding: Прогоняем кроп области глаз через нейросеть. На выходе получаем вектор (например, 512 чисел).

Fusion: Для MVP просто конкатенируем векторы левого и правого глаза в один длинный вектор.

Этап 3: База данных и логика сравнения
Storage: В ChromaDB хранятся векторы разрешенных пользователей с их именами.

Search: Для каждого нового кадра вычисляем косинусное сходство (Cosine Similarity) текущего вектора с векторами из базы.

Threshold (Порог): * Если Distance < Threshold (например, 0.4) — выводим "SUCCESS: [Name]" зеленым цветом.

Если Distance >= Threshold — выводим "ACCESS FORBIDDEN" красным цветом.

3. Пошаговый план реализации
Setup Environment: Установка torch, onnxruntime-gpu, mediapipe, chromadb.

Script 1 (Enrollment): Скрипт для "регистрации". Ты смотришь в камеру, скрипт делает 5–10 кропов глаз, усредняет их эмбеддинги и сохраняет в ChromaDB под твоим именем.

Script 2 (Inference Loop): Основной цикл:

Чтение кадра.

Поиск глаз (MediaPipe).

Получение вектора (InsightFace).

Поиск в ChromaDB.

Отрисовка результата поверх видео.
